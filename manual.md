### 3.4 프로젝트 DB

| 속성 | 타입 | 설명 |
|---|---|---|
| 이름 | title | 프로젝트명 |

| 담당자 | people | 담당자 |
| 날짜 | formula | 자동 계산 기간 |
| 브랜드 | relation | 전체 브랜드/BM DB (독립 업무용) |

- **🎯 목표 연동**: KPI가 연결된 프로젝트 → 브랜드 자동 추출

## 4. 설계 아키텍처

### 4.1 OKR + PM + 재무 삼위일체

```
┌─────────────────────────────────────────────────────────┐
│                    OKR (목표 관리)                        │
│                                                         │
│  라이프타임 GOAL ─→ 연간 경영 목표 ─→ 브랜드/제품 KPI    │
│                                          │              │
│                                     월별 성과 DB         │
│                                          ▲              │
├─────────────────────────────────────────────────────────┤
│                    PM (프로젝트 관리)                     │
│                                                         │
│  KPI ─→ 프로젝트 DB ─→ 세부 업무 DB                     │
│              │         (선행↔후속 의존성)                 │
│              │              │                           │
├─────────────────────────────────────────────────────────┤
│                    재무 (비용 추적)                       │
│                                                         │
│  세부 업무 ─→ Finance DB                                │
│  (정산 그룹 formula로 브랜드 자동 귀속)                   │
└─────────────────────────────────────────────────────────┘
```

### 4.2 Relation 체인 (데이터 흐름)

```
📅 연간 경영 목표 DB
       │
       ▼
📦 브랜드별 제품 KPI ──── 📊 월별 성과
  (상위↔하위 self-rel)        ▲
       │                     │
       ▼                     │
프로젝트 DB ──── 📡 채널 성과 DB
   │
   ▼
세부 업무 DB ──── Finance DB
(선행↔후속)

※ 거의 모든 DB가 [전체 브랜드/BM DB]를 중앙 허브로 참조
※ 프로젝트/세부업무의 '브랜드' 속성은 formula로 KPI→브랜드 자동 추출
```

### 4.3 핵심 설계 원칙

| 원칙 | 설명 |
|---|---|
| **SSoT** | 전체 브랜드/BM DB가 중앙 허브로 단일 진실 공급원 역할 |
| **자동 집계** | Rollup + Formula로 수동 입력 최소화 |
| **맥락 추적** | Finance → 업무 → 프로젝트 → KPI → 브랜드 전체 경로 추적 |
| **의존성 관리** | Self-relation으로 선행/후속 작업 관계 표현 |
| **브랜드 자동 추출** | Formula가 KPI → 브랜드 relation 체인을 자동 추출 |

## 5. 주요 기능별 가이드

### 5.1 WBS (Work Breakdown Structure) 시스템

#### WBS란?

제품 런칭에 필요한 모든 작업을 체계적으로 분해하고, 각 작업의 기간·순서·담당자를 관리하는 시스템입니다.

#### WBS 적용 프로세스

1. **WBS 태스크 정의** → 전체 WBS DB에 기록
2. **AI 프롬프트 실행** → Notion 자동 적용
3. **프로젝트 자동 생성** → 트랙별(제조/마케팅/공통/임상) 분리
4. **세부 업무 자동 생성** → 날짜·의존성 자동 설정

#### 트랙 구성

| 트랙 | 역할 |
|---|---|
| 공통 | 레퍼런스 조사, 제품 형태·성분 결정, 브랜드명·컨셉 확정 |
| 제조 | 제조사 컨택, 샘플 확정, 발주, 제품 제조, QC, 입고 |
| 마케팅 | 상세페이지, 자사몰 세팅, 바이럴, CS/4PL, 판매채널, 마케팅채널 |
| 임상 | 임상/실험 (독립 진행, 약 30일) |

#### 적용 예시 (주름 앰플 런칭)

- **프로젝트 4개**: 주름 앰플 런칭 - 제조/마케팅/공통/임상
- **세부 업무 19개**: 2026-03-01 ~ 2026-04-21 (52일)
- **의존성 18개**: 선행/후속 관계 모두 자동 설정

#### WBS AI 프롬프트 규칙

- KPI 연결: **선택사항** (목표 연동 또는 독립 업무)
- 프로젝트명 형식:
  - KPI 연동: `"[KPI명] - [트랙명]"`
  - 독립 업무: `"[브랜드명] - [트랙명]"`
- Step 2 확인 항목: KPI 선택, 브랜드(조건부 필수), 시작일, WBS DB

| 유형 | 조건 | 브랜드 귀속 방식 |

| 🎯 목표 연동 | KPI 연결됨 | KPI → 브랜드 자동 추출 |

```

세부 업무 없는 경우 → "🏢 전사 공통/운영"
```

**설정 파일** (`mcp_config.json`):
```json

  "mcpServers": {
    "notion": {
      "command": "npx",
      "args": ["-y", "@notionhq/notion-mcp-server"],
      "env": {
        "OPENAPI_MCP_HEADERS": "{\"Authorization\": \"Bearer YOUR_TOKEN\", \"Notion-Version\": \"2022-06-28\"}"
      }
    }
  }
}
```

**주의사항**:
- Node.js 16+ 필요 (18+ 권장)
